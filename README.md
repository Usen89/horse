# 🐎 Табун-Реестр

### Система управления коневодческим хозяйством

Профессиональный зоотехнический учёт: реестр лошадей, косяки и табуны, родословное древо, вакцинация, откорм и архив забоя.

## О проекте

**Табун-Реестр** — веб-приложение для управления табунным коневодческим хозяйством (на примере хозяйства «Алтын Тулпар»). Приложение помогает зоотехникам и владельцам вести полный учёт поголовья с учётом традиционной казахской возрастной классификации лошадей (Құлын, Жабағы, Тай, Байтал, Құнан, Дөнен, Бие, Айғыр).

Все данные хранятся **локально в браузере** (localStorage) — приложение работает офлайн, без сервера и без передачи данных наружу.

## Возможности

| Модуль | Назначение |
| --- | --- |
| 📊 **Главная панель** | Сводная статистика по хозяйству: поголовье, приплод, вакцинация, откорм |
| 🗂️ **Реестр лошадей** | Карточки животных: масть, дата рождения, пол, владелец, статус, фото |
| 👥 **Косяки и табуны** | Группировка лошадей по косякам во главе с жеребцом-производителем |
| 🍽️ **Группа откорма** | Учёт интенсивного откорма, контроль привесов |
| 👶 **Маточное поголовье** | Кобылы, беременность, ожидаемая и фактическая выжеребка, приплод |
| 🛡️ **Вакцинация** | График прививок, сроки ревакцинации, контроль просрочек |
| 💀 **Архив забоя (согым)** | Учёт забоя: вес, выход мяса, выручка |
| 🌳 **Родословное древо** | Визуализация происхождения по отцу (сир) и матери (дам) |
| 📜 **История хозяйства** | Хронология ключевых событий |
| 👤 **Администраторы** | Несколько профилей (зоотехник, ветеринар) с переключением сессий |

## Технологии

- **React 19** + **TypeScript**
- **Vite 6** — сборка и dev-сервер
- **Tailwind CSS 4** — стилизация
- **lucide-react** — иконки
- **motion** — анимации

## Запуск локально

**Требования:** [Node.js](https://nodejs.org/) 18+

```bash
# 1. Установить зависимости
npm install

# 2. (Опционально) указать ключ Gemini API в .env.local
#    cp .env.example .env.local  и вписать GEMINI_API_KEY

# 3. Запустить dev-сервер
npm run dev
```

Приложение откроется на `http://localhost:3000`.

### Прочие команды

```bash
npm run build    # production-сборка в dist/
npm run preview  # локальный просмотр собранной версии
npm run lint     # проверка типов (tsc --noEmit)
```

## Данные и приватность

Хозяйственные данные не покидают устройство — они хранятся в `localStorage` вашего браузера. Для переноса на другое устройство или создания резервной копии используйте функцию **Экспорт / Импорт данных** в приложении.
